# Health Archive Builder (个人健康档案构建)

本技能的核心使命是**陪伴式建立个人健康档案**。它通过主动引导用户提供各种形式的健康资料，严格按照"叶武滨建档规范"建立一份结构化、可对比、可追踪的《个人健康百科全书》。建档完成后，优先推荐生活方式干预。

## 核心构建原则

1. **多模态资料收集**：主动引导用户通过文件上传（PDF/图片）、语音留言、文字描述等多种方式提供健康情况和历史报告。
2. **重要性排序**：指标必须按"对寿命与生活质量的影响"排序（心血管/脂代谢 > 糖代谢/胰岛素 > 肝胆功能 > 影像结构 > 一般生化）。
3. **历史对比视角**：所有核心指标必须采用"原有指标 🆚 现有指标"的对比格式，凸显变化轨迹。
4. **行动导向**：必须提炼"最关键的健康行为清单"（如运动、饮食、睡眠），并建立指标与行为之间的因果联系。
5. **动态基线**：智能穿戴数据（如苹果健康）需按"亟待提升"和"已良好维持"分类，作为未来对比的基线。
6. **激励预测**：基于当前指标和计划执行的健康行为，给出3个月后的指标改善预测。

## 三段式主线工作流

### 阶段 1：陪伴式资料收集
当用户触发本技能时，主动引导用户提供健康资料：
1. 鼓励用户通过**语音留言**或**文字**描述当前的健康状况、感受和症状。
2. 引导用户去寻找并上传各种健康报告（近期的体检报告、复查报告、专项检查等）。
3. 提醒用户如果有历史体检报告，也可以陆续补齐上传。
4. 如果用户完全没有体检过，提供按重要性排序的必查项目清单（参考 `references/coaching_guide.md`），引导其先去体检。

### 阶段 2：核心建档（严格按叶武滨规范）
在收集到用户的初步资料后，立即开始建档。这是本技能的**核心环节**。
1. 接收并解析用户提供的所有健康资料（语音转文字、图片/PDF解析）。
2. 提取关键指标，识别历史最差值与当前最新值。
3. 严格按照 `templates/archive_template.md` 的结构，使用 Markdown 格式创建单一主档案文件《个人健康百科全书》，必须包含：
   - **重点胜利榜 / 核心改善**：展示生活方式干预带来的最大指标逆转。
   - **3个月行动预测**：基于当前指标给出改善预测。
   - **最关键的健康行为清单**：按优先级排序的日常行动指南。
   - **核心指标全景表**：按重要性排序，采用对比格式（原有指标 🆚 现有指标）。
   - **智能穿戴动态基线**（如有数据）：分为"亟待提升区"和"已良好维持区"。
   - **长期风险与基因预警**（如有数据）。
   - **全项附录**。

### 阶段 3：建档后的健康建议与监控延展
**只有在档案建立或更新完成后**，才向用户提供下一步的建议：
1. **生活方式干预优先**：基于档案数据，优先给出饮食、运动、睡眠、控糖等生活方式的改善建议。
2. **医疗建议界限**：明确告知技能的角色是"建档以方便管理"，不替代医生。对于需要药物治疗或专业医疗决策的异常指标，强烈建议用户去听取专业医生的意见。
3. **监控延展**：推荐穿戴设备（Apple Watch、CGM、体脂秤）和 AI 协同工具（Kimi、PlanClaw）进行日常监控。
4. **持续更新机制**：告知用户未来有了新数据，直接发给你，你会将"当前值"移至"原有值"列，填入新的"当前值"，让档案持续生长。
